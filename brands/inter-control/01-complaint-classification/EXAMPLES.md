## 示例 1：菜鸟驿站破损 + 态度恶劣（P1）

**用户输入：**
```
今天去菜鸟驿站拿快递，取到手发现快递盒子已经被压坏了，里面装的是两件新衣服（京东下单，原价328元），衣服有污渍。站长说这不是他们的责任，还叫我去找商家，语气很冲。我觉得明显是驿站保管不善，怎么投诉？
```

**Skill 输出：**
```
【投诉智能分类结果】

📋 基本信息
─────────────────────────────
• 业务线：菜鸟驿站
• 投诉来源：用户自述
• 提交时间：未知

🏷️ 分类结果
─────────────────────────────
• 一级分类：F. 驿站专项 + C. 售后服务
• 二级分类：B3. 货物破损 + C6. 态度恶劣（推诿）
• 三级标签：[资损风险][驿站违规]

⚠️ 优先级
─────────────────────────────
• 级别：P2 中
• 判定依据：货值328元（100-1000区间），态度差但未达辱骂/肢体冲突级别

👤 责任方判断
─────────────────────────────
• 初步责任方：驿站（需核实）
• 判断依据：
  ① 外包装破损发生在驿站保管期间（轨迹到站后未出库期间）
  ② 站长推诿行为不符规范（代收服务有赔付义务）
  需驿站在18小时内提交：监控视频（包裹接收时状态）+ 入库记录

🔀 派发建议
─────────────────────────────
• 建议派发至：驿站合规团队
• 建议处理时限：24小时
• 处理路径：
  1. 通知驿站提交举证（18小时内）
  2. 核实包裹入库时照片/视频证明
  3. 若责任确认在驿站：启动赔付流程
     - 破损赔付：按货物残值与328元差额，最高不超过328元
  4. 站长推诿行为记入驿站服务评分

📎 备注
─────────────────────────────
• 衣服类商品建议用户提供购买凭证（京东下单截图）以核实货值
• 若用户同时在京东投诉，需同步协调避免双重赔付
```

## 示例 2：物流丢件 + 高价值（P1→潜在P0）

**用户输入：**
```
订单号：CN2024031523456
三周前寄出一台笔记本电脑（价值7800元），菜鸟快递显示"已到达上海转运中心"之后就再也没有更新了，已经超过21天。商家说已发货，快递说联系不上。我急需这台电脑工作，请问怎么处理？
```

**Skill 输出：**
```
【投诉智能分类结果】

📋 基本信息
─────────────────────────────
• 业务线：菜鸟物流
• 投诉来源：用户自述（含订单号）
• 提交时间：未知（事发约21天前）

🏷️ 分类结果
─────────────────────────────
• 一级分类：B. 物流配送
• 二级分类：B2. 丢件（入库后丢失）+ B6. 轨迹停滞
• 三级标签：[资损风险]（货值7800元，超高价值）

⚠️ 优先级
─────────────────────────────
• 级别：P0 紧急 ⚠️
• 判定依据：投诉金额7800元，远超5000元P0触发阈值；轨迹停滞21天（极度异常）

👤 责任方判断
─────────────────────────────
• 初步责任方：物流商（菜鸟快递）
• 判断依据：
  ① 轨迹最后更新为"到达上海转运中心"，此后无任何操作记录（21天）
  ② 属于"入库后丢失"场景，物流商承担举证责任
  ③ 商家已发货且有揽收记录，商家责任排除

🔀 派发建议
─────────────────────────────
• 建议派发至：内控负责人（P0必须知晓）+ 物流商务合作团队
• 建议处理时限：2小时内启动，4小时内给用户初步回复
• 处理路径：
  1. 【立即】调取订单 CN2024031523456 上海转运中心操作记录
  2. 【2小时内】联系物流商上海中心核查包裹去向
  3. 【4小时内】给用户初步回复（正在处理，告知预期时间）
  4. 确认丢失后：启动赔付流程（按实际价值，含运费）
  5. 电子设备7800元超标准赔付上限，需核实是否有保价记录
  
⚠️ 特别提示：
- 电子设备（笔记本）属于高价值易失窃物品，若确认丢失应同步核查该批次/该中心是否有同类案件
- 建议调取用户近期同类投诉，判断是否为系统性问题

📎 备注
─────────────────────────────
• 用户反映有工作紧迫性，建议沟通时说明处理进展，降低舆情风险
```

## 示例 3：淘宝闪购虚假宣传（P2）

**用户输入：**
```
在淘宝闪购买了一款面霜，商家宣传"7天见效，美白祛斑"，我用了一个月完全没效果，感觉就是骗人的。而且产品包装上的成分表里没有写任何美白成分。要投诉商家。
```

**Skill 输出：**
```
【投诉智能分类结果】

📋 基本信息
─────────────────────────────
• 业务线：淘宝闪购
• 投诉来源：用户自述
• 提交时间：未知（使用约1个月）

🏷️ 分类结果
─────────────────────────────
• 一级分类：D. 虚假宣传 + A. 商品质量
• 二级分类：D2. 夸大宣传 + A3. 不符合强制标准（待核实）
• 三级标签：[商家违规][法律风险]

⚠️ 优先级
─────────────────────────────
• 级别：P2 中
• 判定依据：涉及化妆品夸大功效宣传，属典型D2；成分表与宣传不符可能涉及《广告法》违规

👤 责任方判断
─────────────────────────────
• 初步责任方：商家
• 判断依据：
  ① 商家宣传页使用"7天见效、美白祛斑"等绝对化表述
  ② 产品成分表与宣传功效不符（无美白成分）
  → 涉嫌违反《广告法》第28条（虚假广告）

🔀 派发建议
─────────────────────────────
• 建议派发至：市场合规团队（虚假宣传专项）
• 建议处理时限：24小时
• 处理路径：
  1. 调取商家商品页面截图（宣传内容存档）
  2. 核对产品成分表与宣传的一致性
  3. 如确认违规：按平台商家违规规则处理（下架商品+警告/处罚）
  4. 用户退款：建议支持7天无理由退款申请（依《消费者权益保护法》）
  5. 如商家成分虚假且涉及安全（如含禁用成分），升级P1并通报监管

📎 备注
─────────────────────────────
• 化妆品"美白""祛斑"属于特殊功效，依《化妆品监督管理条例》需取得特殊化妆品注册
• 建议同时核查该商家是否有特殊化妆品注册证
• 若该商家有大量同类投诉，需纳入商家风险评分
```

## 示例 4：批量分析模式（5条投诉清单）

**用户输入：**
```
帮我分析一下今天收到的5条投诉：
1. 驿站站点A：用户说取件时被要求额外付2元"保管费"
2. 驿站站点A：另一个用户也说被收了1.5元
3. 驿站站点A：第三个用户说被收了"扫码取件费"3元
4. 供应商X：物流延误，预计3天到但第8天才到
5. 供应商Y：包裹破损，里面手机屏幕碎了（价值3200元）
```

**Skill 输出：**
```
【投诉批量分析报告】

📊 总体概况
─────────────────────────────
• 总条数：5条
• 业务线分布：菜鸟驿站（3条）、菜鸟物流（2条）

📈 分类分布
─────────────────────────────
| 一级分类 | 数量 | 占比 |
| F3. 额外收费（驿站） | 3 | 60% |
| B1. 时效延误 | 1 | 20% |
| B3. 货物破损（高价值） | 1 | 20% |

⚠️ 优先级分布
─────────────────────────────
• P0紧急：0条
• P1高：1条（供应商Y破损，货值3200元）
• P2中：4条（延误1条 + 额外收费3条）
• P3低：0条

🔥 异常发现 ⚠️ 重要
─────────────────────────────
• 【系统性违规警报】驿站站点A 在同一天出现3条额外收费投诉（1.5元/2元/3元），
  且收费名目各异（保管费/扫码取件费），疑似驿站站长主动设置违规收费项目
• 建议立即合并为一个专项案件处理，触发驿站合规核查

📋 TOP3 高风险投诉（需优先处理）
─────────────────────────────
1. 【P1 供应商Y】手机屏幕破损，货值3200元
   → 立即调查物流环节责任方，启动赔付流程，处理时限：8小时
   
2. 【P2 驿站站点A - 系统性额外收费】3条合并处理
   → 派驿站合规团队核查站点A收费规则，收集用户凭证
   → 核查是否达到清退条件（投诉成立率/严重违规）
   → 处理时限：24小时
   
3. 【P2 供应商X - 延误8天】
   → 核实SLA承诺，调取物流时效记录，处理时限：24小时

🔀 派发汇总
─────────────────────────────
| 部门 | 待处理数量 | 紧急程度 |
| 驿站合规团队 | 3条（合并为1件） | P2，24h内 |
| 物流商务合作团队 | 2条 | P1/P2，8-24h |

⚠️ 特别提醒：
驿站站点A 的3条投诉一旦经核实成立，需同步评估：
- 投诉成立率是否触发清退预警（≥0.5%连续2月）
- 额外收费行为是否属于严重违规（超收按2倍赔偿）
- 是否需要立即下架该站点
```

## 示例 5：信息不足，引导补充

**用户输入：**
```
我买的东西有问题，很愤怒，想投诉
```

**Skill 输出：**
```
【需要补充信息】

为了准确分类和帮你推进处理，需要了解以下几点（一次性回复即可，没有的可以跳过）：

1. 是哪个业务的问题？
   → 菜鸟物流（快递）/ 菜鸟驿站（取件点）/ 淘宝闪购（商品）/ 其他？

2. 具体发生了什么？
   → 商品质量？物流问题（丢件/破损/延误）？客服/售后问题？

3. 有没有订单号或商品信息？（有的话附上）

4. 大概金额是多少？

补充后我会立即给出分类结果和处理建议。
```
