## 一、快速判断卡片（QUICK REFERENCE）

```
输入一条投诉 → 扫描关键词 → 匹配规则 → 输出分类+优先级
```

### 关键词→分类映射

| 关键词/场景 | 一级分类 | 优先级触发 |
|------------|---------|----------|
| 丢件、未收到、消失、查无此件 | B2丢件 | P1（单件）/ P0（批量/高价值） |
| 破损、碎了、压坏、包装坏 | B3破损 | P1（货值>500）/ P2（一般） |
| 延误、超时、好几天没动静 | B1延误 | P2（一般）/ P1（偏远+超7天） |
| 虚假签收、显示签收但没收到 | B4虚假签收 | P1 |
| 辱骂、态度差、骂人、动手 | C6态度恶劣 | P1（辱骂）/ P0（动手+伤人） |
| 不退款、拒绝退款、退款超时 | C1退款难 | P1（超7天）/ P2（未超时） |
| 假货、假的、仿冒品 | A1假冒伪劣 | P1 |
| 乱收费、多收钱、强制收费 | F3额外收费 | P2 |
| 个人信息泄露、隐私、手机号被卖 | G1信息泄露 | P0（>100人）/ P1（个人） |
| 媒体、曝光、微博、抖音、热搜 | 任意+[媒体风险] | P0 立即处理 |
| 刷单、虚假签收出库 | F6刷单 | P0（组织性）/ P1（单次） |

## 二、分类判断决策树

### 2.1 物流类（B类）

```
投诉涉及快递/包裹？
├─ YES → 是否已揽收？
│         ├─ 已揽收 → 是否到达驿站/目的地？
│         │           ├─ 到达驿站 → 驿站问题（F类）
│         │           └─ 未到达 → 物流中转问题（B类）
│         └─ 未揽收 → 商家发货问题（商家责任）
└─ NO → 非物流投诉，继续其他分类
```

**轨迹判断节点：**
- 揽收 → 转运 → 到达目的分拨 → 到达驿站/派送网点 → 派送 → 签收
- 问题发生在哪个节点，即哪段负责

### 2.2 驿站类（F类）

```
问题发生在驿站？
├─ 包裹在驿站期间丢失/破损 → F类 + B类 双标
├─ 驿站行为不当（收费/拒收/态度） → F类
├─ 驿站系统问题（虚假出库/信息错误） → F类
└─ 驿站只是转运方（中转丢件） → B类 物流商责任
```

### 2.3 商品类（A类）

```
商品本身有问题？
├─ 外观/功能与描述不符 → A2以次充好
├─ 怀疑是假货（有品牌） → A1假冒伪劣
├─ 无生产日期/生产厂家/合格证 → A3不符合强制标准
└─ 食品/化妆品过期 → A4过期变质（P0/P1）
```

## 三、优先级升级规则

**以下情况，在基础优先级基础上升一级：**

| 升级条件 | 基础→升级 |
|---------|----------|
| 涉及孕妇/老人/儿童 | P2→P1 |
| 投诉人自称记者/律师/人大代表 | P2→P1，P1→P0 |
| 投诉内容已截图传播（有链接证据） | P2→P1，P1→P0 |
| 同一驿站/商家同类投诉≥3条/48小时 | P2→P1，P1→P0 |
| 节假日（春节/618/双11）期间 | P3→P2，P2→P1 |
| 投诉金额临界（接近下一级阈值±10%） | 按高一级处理 |

## 四、责任方判断规则库

### 4.1 物流商责任
**确认依据（满足任一）：**
- 轨迹显示到达物流商仓库后无更新 > 72小时
- 物流商仓库操作记录显示入库后丢失
- 物流商工作人员自述操作失误
- 比对重量/包裹照片发现物流环节破损迹象

### 4.2 驿站责任
**确认依据（满足任一）：**
- 轨迹到站后，驿站未在规定时间内通知用户
- 用户在驿站现场取件时发生破损（需监控视频核实）
- 驿站工作人员明确收取额外费用（有收据/聊天记录）
- 驿站人员辱骂用户（有录音/视频）
- 系统显示已出库但用户未取件（疑似虚假出库）

### 4.3 商家责任
**确认依据（满足任一）：**
- 商品描述页面与实物有明显不符（有截图对比）
- 商家在退款/退货申请提交后7天内无响应
- 商家聊天记录中明确拒绝或推诿
- 商品质量问题发生前，物流轨迹正常

### 4.4 用户自身责任
**确认依据（满足任一）：**
- 用户签收后外包装完好，开箱发现问题（需举证）
- 用户超过平台规定的退款期限后才投诉
- 用户操作失误（填错地址/联系方式等）
- 用户申报价值与实际明显不符（影响赔付金额）

### 4.5 平台责任
**确认依据（满足任一）：**
- 平台规则文字本身有歧义或矛盾
- 平台系统错误导致订单异常
- 平台客服处理流程导致问题扩大

## 五、赔偿金额快速参考

### 菜鸟驿站（代收服务）
| 情形 | 赔付标准 |
|-----|---------|
| 货物丢失（有货值证明） | 最高 **500元** |
| 货物破损 | 最高不超过**订单成交价** |
| 延迟代收 | **5元/单** + 损失差额 |
| 拒收包裹 | **5元/单** + 损失差额 |
| 多收费用 | 多收金额 × **2倍** |
| 辱骂消费者 | **500元/单** |
| 地图定位错误 | **5元/单** |

### 菜鸟驿站（代揽件服务）
| 情形 | 赔付标准 |
|-----|---------|
| 货物丢失（有货值） | 全额赔付，上限 **1000元** + 运费 |
| 货物破损（部分） | 最高货值20%，上限 **300元** |
| 代寄延时 | **5元/单** |

### 菜鸟专线（跨境）
| 情形 | 赔付上限 |
|-----|---------|
| 揽收后丢失/破损 | **300元/包裹** |
| 重量不符 | 退还多付运费 |

## 七、特殊场景处理 SOP

### 7.1 批量投诉（疑似组织行为）

**触发条件：** 同类投诉≥5条/24小时，且投诉内容高度相似

**处理步骤：**
1. 立即升级为P0
2. 保存所有原始投诉内容
3. 提取关键相似词句，判断是否出自同一模板
4. 联系用户样本（随机3-5个）核实真实性
5. 通知内控负责人和法律团队
6. 如确认组织行为，走反欺诈流程（非普通投诉流程）

### 7.2 媒体曝光风险

**触发条件：** 投诉用户提供了微博/抖音/媒体链接，或自称是记者

**处理步骤：**
1. 立即升级为P0
2. 通知公关团队和内控负责人
3. 在正常处理流程基础上，追加PR审查步骤
4. 对外回复统一由公关团队审稿

### 7.3 驿站清退判断

**触发条件：** 投诉涉及某驿站站点，核实后发现：

```
判断清退可能性：

STEP 1：查连续2月投诉成立率
  IF 投诉成立率 ≥ 0.5% 连续2月 → 进入清退流程

STEP 2：查严重违规行为
  IF 有辱骂/殴打/刷单/负面媒体报道 → 立即清退

STEP 3：查业务量
  IF 上线>91天 且 近30天提货成功 < 15单 → 业务不达标清退
```
