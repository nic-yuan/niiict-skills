## 对话示例

### 示例1：单条丢件查询

**用户输入：**
> 帮我看一下这个订单，好像丢了：
> 运单号：CN2025041001234
> 下单时间：4月1日
> 承诺3天到货（省内）
> 最后轨迹：4月1日22:30 已揽收（广州网点）
> 之后再没更新了，现在4月10日了

**Skill 输出：**
```
【物流异常预警报告 - 单件】

📦 订单信息
─────────────────────────────
• 订单号/运单号：CN2025041001234
• 业务线：菜鸟物流（境内）
• 货值：未提供（需补充）
• 下单时间：2025年4月1日
• 承诺时效：3天（省内）→ 应于4月4日到达
• 当前状态：4月1日22:30 揽收后无更新（停滞9天）

🚨 异常识别结果
─────────────────────────────
• 异常类型：AE-04 疑似丢件 + AE-02 超时未达
• 详情：
  - 揽收后超72小时无入库记录（实际停滞约9天）
  - 承诺3天到货，已超时6天，超过平台5天违规触发点

⚠️ 预警等级
─────────────────────────────
• 级别：P1 高级预警 ⚠️⚠️
• 响应时限：8小时内介入
（如货值>5000元，升级为P0）

💰 赔偿预算估算
─────────────────────────────
• 适用规则：
  ① 物流时效超时赔付（淘天规则2024版）
  ② 入库后丢失赔付（菜鸟驿站代收规则）

• 赔付预估（待确认货值后精算）：
  | 场景 | 预计赔付 |
  | 确认丢件（有货值证明） | 货值全额，最高500元/1000元 |
  | 仅时效超时（未确认丢失） | 订单金额×5%，最低5元，最高50元 |

• ⚠️ 缺失货值信息，无法精确计算，请补充申报金额

📋 建议处置措施
─────────────────────────────
• 立即行动（8小时内）：
  1. 联系菜鸟物流调取广州揽收网点的处理记录
  2. 核实4月1日22:30后是否有分拨操作但未更新轨迹
  3. 请用户补充：货值申报金额、是否有保价记录
  
• 预计处置周期：
  - 供应商反馈：72小时内（依规则）
  - 如确认丢件：启动赔付，处理时限参照P1（24小时内给用户回复）
  
• 复查节点：今日行动后24小时，确认调查进展

📎 备注
─────────────────────────────
• 省内9天未到属于严重异常，即使无丢件也应追究时效责任
• 投诉时效从运单创建起算，该件仍在有效期内（90-120天）
• 如供应商无法提供入库记录，默认视为揽收后丢失
```

### 示例3：赔偿金额计算

**用户输入：**
> 有个投诉要处理赔偿，菜鸟专线，包裹在途丢失了，买家申报货值500美元，有上传申报单，请计算赔付金额

**Skill 输出：**
```
【赔偿金额计算】

📦 订单信息
─────────────────────────────
• 服务类型：菜鸟专线（跨境）
• 异常类型：在途丢失（入库后丢失）
• 申报货值：$500 USD

💰 赔偿计算
─────────────────────────────
适用规则：菜鸟专线《投诉举证和赔付标准》- 入库后丢失

规则说明：
• 按实际损失价值赔偿（以订单申报金额为准，扣除支付/服务手续费）
• 最高不超过 300元人民币/包裹

赔偿计算：
• 申报货值：$500 USD
• 当前参考汇率（实际以赔付月份第一日汇率为准）
  → 约合：¥3,600 RMB（估算）
  
→ 申报金额（¥3,600）超出赔付上限（¥300）
→ 实际赔付金额：¥300 RMB（封顶）

📋 赔付执行说明
─────────────────────────────
• 赔付方式：菜鸟在物流商保证金或代收物流费中直接扣减
• 汇率确认：以赔付生成当月第一个自然日汇率为准
• 举证要求：买家已提交申报单，举证成立
• 处理时限：物流商10个自然日内反馈 → 未达成可在15天内申诉

⚠️ 提示
─────────────────────────────
• 菜鸟专线最高赔付上限为300元RMB，与货值无关（货值超出上限部分不赔）
• 如买家有强烈异议，建议内控负责人酌情处理（平台商誉考量）
• 建议提醒买家：跨境高价值货物建议使用保价服务（保价金额另议）
```

## 特殊情况处理

### 偏远地区异常

**判断标准：** 新疆、西藏、内蒙古、海南（部分）、边境地区

**处置原则：**
- 时效标准宽松（7-14天正常）
- 轨迹停滞预警时间延长至120小时（5天）
- 丢件赔偿标准不变
- 优先核实当地运输条件，不轻易认定丢件

### 春节/双11/618大促期间

**判断标准：** 平台公告的特殊时效保护期

**处置原则：**
- 时效超时不触发违规（平台有特殊规则）
- 但超过保护期后，恢复正常时效标准
- 如用户已明确有损失，不受特殊期保护

### 批量同类异常应急处置

**触发标准：** 同供应商/批次≥5件 / 24小时内

**应急SOP：**
```
1. 立即升级P0，通知内控负责人
2. 向供应商发出正式查询（书面留痕）
3. 核实当地是否有突发事件（天气/事故/系统）
4. 48小时内评估是否需要转承运
5. 向受影响用户发主动告知（先于用户投诉）
6. 记录入供应商风险档案，触发风险评分复评
```

## 注意事项

1. **预警是为了主动处置，不是等用户投诉后才行动**。发现异常应主动联系承运商和用户，不要等48小时。
2. **赔偿金额是参考**，最终以平台规则原文为准，平台规则更新后本Skill需同步更新。
3. **批量异常优先级最高**，不要因为单件金额小而忽略批量信号。
4. **记录每次异常**，异常数据纳入供应商风险评分（联动风险评分Skill）。
5. **本Skill适用于内控人员**，对外沟通（向用户/供应商）需另行起草正式回复，不直接使用本报告内容。
