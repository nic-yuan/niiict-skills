## 注意事项

1. **评分是辅助工具，不是最终判决**。一票否决情况须人工复核。
2. **数据质量决定评分质量**。如供应商自报数据与平台数据不符，以平台数据为准。
3. **每个维度的基准数值来自行业公开数据**，如业务场景特殊，可由内控负责人调整权重。
4. **清退是最后手段**。处置建议按"辅导→预警→限量→清退"递进，不要跳过中间步骤。
5. **本Skill不对外（供应商/站长）公示评分细节**，内容仅供内控人员内部使用。

## 历史记录有效期（v1.2 新增）

> 详细规则见 `docs/GLOSSARY.md` 第八节

| 违规类型 | 有效期 | 过期后处理 |
|---------|--------|----------|
| 供应商轻微违规 | 12个月 | 降一级计算（严重→轻微→清除） |
| 供应商严重违规（批量丢件/媒体曝光） | **永久** | 不清零，永为记录 |
| 驿站轻微违规 | 12个月 | 不计入评分 |
| 驿站严重违规（辱骂/刷单/媒体） | **永久** | 一票否决，永不清零 |
| 投诉成立记录 | 12个月 | 用于判断累犯 |

> ⚠️ **跨期计算：** 如供应商在第13个月再次违规，将与第1个月的违规分别计算，不视为连续累犯。

## 试用期评估规则（v1.2 新增）

### 新供应商试用期（上线前3个月）

```
评估规则：
1. 不适用正常KPI考核标准（阈值放宽）
2. 但记录所有异常事件（异常件、投诉成立等）
3. 第3个月结束时，一次性评估试用期表现
4. 试用期评估结果决定是否转正

试用期转正标准：
├─ 无严重违规记录 → 转正，按正常标准考核
├─ 有1-2次轻微违规 → 转正，降低业务量分配20%
├─ 有3次以上轻微违规 → 延长试用期1-3个月
└─ 有任何严重违规 → 立即终止合作（不转正）
```

### 新驿站试用期（开业前3个月）

```
评估规则：
1. 投诉成立率标准放宽至1%（正常0.5%的2倍）
2. 但任何一次严重违规（辱骂/刷单）即触发立即清退
3. 出库率要求：≥80%（正常≥85%的放宽版）
4. 第3个月结束时综合评估
```

## 跨业务线评分规则（v1.2 新增）

### 同一供应商同时服务物流和驿站

```
场景：某物流供应商同时为菜鸟物流和驿站提供服务

评分方式：分开评分

├─ 物流服务评分（模型A）：独立计算
├─ 驿站服务评分（模型B）：独立计算
└─ 综合风险：取两个评分的最低值

示例：
├─ 物流评分：75分（🔵关注级）
└─ 驿站评分：52分（🟠高风险）
→ 综合风险等级：🟠高风险（取最低）
→ 建议：优先处理驿站服务问题，但物流服务也需关注

决策规则：
├─ 任一评分 < 40分 → 暂停该业务线合作
├─ 两个评分均 < 40分 → 暂停全部合作
└─ 任一评分触发一票否决 → 立即终止该业务线合作
```

## 趋势分析模式（v1.1 新增）

当用户提供**连续3个月以上**的历史数据时，Skill 进入趋势分析模式：

### 趋势判断规则

```
评分趋势：
连续3月下降 → 🔴 恶化趋势（提前预警，不等红线）
连续3月上升 → 🟢 改善趋势（可减少监控频率）
波动无规律  → 🟡 不稳定（维持当前监控频率）
单月急降>15分 → ⚠️ 急速恶化（立即复评）
```

### 早于红线预警机制

```
以下情况，即使未到清退红线，也提前预警：

驿站：
- 投诉成立率连续上升2月，即使单月<0.5%，也发出"趋势预警"
- 业务量连续下降3月（即使仍>15单），发出"经营风险预警"

物流供应商：
- 时效达成率连续下降且降幅>2%/月，发出"趋势预警"
- 投诉率连续上升且每月增加>0.02%，发出"趋势预警"
```

### 趋势报告补充输出

```
📈 趋势分析（基于[N]个月数据）
─────────────────────────────
| 维度 | 3月均值 | 本月 | 趋势 |
[填充]

⚠️ 趋势警告：[如有]
→ [XX指标]连续[N]月[上升/下降]，
  当前值[X]，预计[N]个月后达到红线[Y]
  建议：[提前介入措施]
```
