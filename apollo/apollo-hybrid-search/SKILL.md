# Apollo Hybrid Search — 混合多源搜索系统

## 核心准则

**AI的搜索应该像多引擎飞机——多个引擎同时工作，但只操控一个驾驶杆。**

MemFree的启发：
- "Hybrid AI Search" — 同时从知识库+全网搜索
- "Automatically determines when to perform internet searches" — 自动判断搜索时机
- 多个搜索源并行，结果智能融合

对应的AI映射：
- **多引擎并行**：内部记忆库 + 外部搜索 + 向量库同时查询
- **自动决策**：判断哪个源的结果更可靠
- **结果融合**：去重+排序+摘要，一次输出

## 与 apollo-neuro 的关系

apollo-neuro（信息路由决策树）负责判断"用哪个源"：
- 内部有答案 → 直接输出
- 需要最新信息 → 外部搜索
- 两者都要 → hybrid-search执行融合

apollo-hybrid-search 负责"怎么融合"：
- 接收 neuro 的路由决策
- 执行多源查询
- 融合结果返回

## 搜索源分类

| 源 | 适用场景 | 优先级 |
|---|---------|--------|
| 内部记忆（memory/MEMORY.md） | 已知事实、个人偏好、历史决策 | 高（首选） |
| 工作区文件（workspace/） | 具体项目上下文、代码、文档 | 高 |
| 向量记忆（tdai_memory_search） | 模糊相关、跨会话关联 | 中 |
| 全网搜索（web_fetch/search） | 最新信息、行业动态、陌生领域 | 低（需要时） |
| 工具API（邮件/日历/飞书） | 实时数据、系统状态 | 低（按需） |

## 融合流程

```
用户查询 →
  1. apollo-neuro 判断搜索策略
     - 内部优先？外部？混合？
  2. 多源并行查询（按neuro决策）
  3. 结果汇集
  4. 去重 + 质量评估
  5. 融合排序
  6. 输出统一答案
```

## 融合策略

### 策略1：内部优先（默认）
```
内部有明确答案 → 直接输出内部结论
内部模糊 → 内部结论 + 外部补充
内部没有 → 外部为主
```

### 策略2：外部验证
```
内部有结论 → 用外部搜索验证
外部一致 → 置信度提升
外部矛盾 → 标注矛盾，保留内部+外部双视图
```

### 策略3：纯粹外部
```
需要最新信息/行业动态 → 外部搜索为主
内部只提供上下文参考
```

## 信息路由决策树（来自 memfree auto-decision）

```
收到查询 →
  1. 查内部memory/文件？ → 有直接相关记录 → 优先内部
  2. 查外部（搜索/API）？ → 需要最新信息 → 直接外部
  3. 混合模式 → 内部结论+外部补充 → 两者都查
  判断依据：任务类型+时间敏感性+历史成功路径
```

## 输出格式

```
🔀 [apollo-hybrid-search] 混合搜索结果

查询：[用户问题]
搜索策略：[内部优先/外部验证/纯粹外部]
融合来源：[memory] + [workspace] + [web]

融合结果：
  来源A：[结论]（置信度：高/中/低）
  来源B：[结论]（置信度：高/中/低）

统一答案：[融合后的最佳回答]

来源标注：
  - 内部记忆（置信度：高）
  - 全网搜索（URL：xxx）
```

## 使用场景

| 场景 | 策略 | 说明 |
|------|------|------|
| 问Nic的偏好/习惯 | 内部优先 | 内部记忆有答案 |
| 问最新行业动态 | 纯粹外部 | 需要实时信息 |
| 复杂问题分析 | 外部验证 | 内部结论+外部验证 |
| 跨会话关联查找 | 混合模式 | 两者都查 |
| 新领域探索 | 外部优先 | 内部无相关知识 |
